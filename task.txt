Django REST Framework

REST - Representation State Transfer

Ресурс - ключевая абстракция протокола HTTP. Ресурс - это все, что программист хочет сделать
доступным для внешнего мира через приложение

1) Ресурс - все авторов
2) Ресурс - конкретный автор

URL в REST

URL - обозначение, к какому ресурсу мы обращаемся
authors/
categories/

Работа с REST заключается в первую очередь в определении ресурсов, которые мы хотим
открыть внешнему миру (то есть создаем URL, которые обозначают эти ресурсы)
После используем HTTP-глаголы для взаимодействия с этими ресурсами

REST реализутся при помощи отдачи данных через общеизвестные форматы
JSON
XML

HTTP - транспорт для данных

Компоненты REST
HTTP определяет следующую структуру запроса:
- строка запроса - определяет тип сообщения GET authors/
- заголовки (headers) - характеризуют тело сообщения, параметры передачи и прочие важные сведения
- тело (body) - необязательное и оно несет в себе любую полезную нагрузку (JSON) POST


HTTP ответ:
- строка состояния (status_code и сообщение о причине)
- заголовки (headers)
- тело (body)

RESTFUL API

1) Единый интерфейс - строится вокруг ресурсов с уникальными URL, а каждая страница реализовывает
какой-либо из HTTTP-методов. Единый интерфейс по сути - протокол HTTP

2) Архитектура Клиент-Сервер - это пользовательский интерфейс сайта или приложения. В Restful
API код запросов остается на стороне клиента, а код для доступа к данным - на стороне сервера.

3) Нестационарные запросы (stateless) - это означает, что каждый HTTP-запрос полностью изолирован.
Сервер не полагется на информацию из предыдущих запросов - клиент отправляет HTTP-запрос, вклющающий
сразу всю необхомую информацию для сервера и сервер не хранит никаким образом контекст запросов.

4) Кэшируемость - в данных запроса должно быть указано, нужно ли кэшировать данные, Если такое указание есть, клиент
получит право обращатся к буферу при необходимость

5) Многоуровневость системы - в RESTful API сервер может быть разделен на много разных уровней, но при этом
каждый сервер взаимодействует только с ближайшим уровнем и не связан с другими

6) Предоставление кода по запросы - Сервера могут отправлять клиенту код (например, скрипт для запуска видео),
Так общий код сайта (фронтенда) становится сложнее только при необхомости
